# Что такое `inline` переменные в C++17?

## Ответ
`inline` переменные - это нововведение C++17, которое позволяет определять переменные в заголовочных файлах без нарушения правила одного определения (ODR - One Definition Rule).

### Основные характеристики inline переменных:

1. **Определение в заголовочных файлах**
   - Можно определять переменные в заголовочных файлах
   - Не нужно использовать extern в заголовочном файле и определение в .cpp файле

2. **Единое определение**
   - Все определения inline переменной должны быть идентичны
   - Компилятор гарантирует, что будет использоваться только одна копия переменной

3. **Инициализация**
   - Должна быть инициализирована при определении
   - Инициализация должна быть константной

## Примеры кода

### Правильное использование:

```cpp
// header.h
inline int globalCounter = 0;
inline const double PI = 3.14159;

class MyClass {
public:
    static inline int instanceCount = 0;
    static inline const std::string defaultName = "Default";
};

// Можно использовать в нескольких .cpp файлах
// file1.cpp
#include "header.h"
void func1() {
    globalCounter++;
    MyClass::instanceCount++;
}

// file2.cpp
#include "header.h"
void func2() {
    globalCounter++;
    MyClass::instanceCount++;
}
```

### Неправильное использование:

```cpp
// Неправильное использование inline
inline int uninitialized;  // Ошибка: inline переменная должна быть инициализирована

// Неправильное определение в разных файлах
// header1.h
inline int value = 42;

// header2.h
inline int value = 43;  // Ошибка: разные значения для inline переменной

// Неправильное использование с неконстантной инициализацией
inline int randomValue = rand();  // Ошибка: инициализация должна быть константной

// Неправильное использование в классах
class BadClass {
    inline int member;  // Ошибка: inline нельзя использовать для членов класса
public:
    static inline int* ptr = new int(42);  // Опасность: утечка памяти
};
```

## Важные замечания
1. `inline` переменные особенно полезны для статических членов классов
2. Все определения inline переменной должны быть идентичны во всех единицах трансляции
3. Инициализация должна быть константной и выполняться во время компиляции
4. `inline` переменные не могут быть членами класса (кроме static)
5. При использовании `inline` переменных с динамической памятью нужно быть осторожным с утечками памяти
